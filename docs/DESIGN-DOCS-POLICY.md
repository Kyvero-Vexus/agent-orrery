# Design Docs Policy

For every bead implemented in this repository, the contributor must also update Common Lisp design docs in:

- `/home/slime/projects/emacsen-design-docs`

Required update content:

1. Problem statement and constraints
2. Typed interface/data model changes
3. Functional architecture notes (pure/impure boundaries)
4. E2E coverage evidence and known gaps

Bead descriptions in this repo must explicitly reference this policy.
